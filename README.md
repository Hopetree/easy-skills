# easy-skills

Claude Code 技能（Skills）合集，提供可复用的 AI 辅助开发工作流。两个技能围绕项目文档形成**「创建 → 审查 → 同步更新」的管理闭环**。

## 文档管理闭环

```
┌─────────────────────────────────────────────────────────┐
│                    项目文档生命周期                        │
│                                                         │
│   project-init-docs          code-review-doc-sync       │
│   ┌──────────────┐           ┌──────────────┐           │
│   │ 了解项目需求   │           │ 确定 review   │           │
│   │              │           │ 范围          │           │
│   └──────┬───────┘           └──────┬───────┘           │
│          ▼                          ▼                    │
│   ┌──────────────┐           ┌──────────────┐           │
│   │ 评估文档清单   │           │ 代码 Review   │           │
│   └──────┬───────┘           └──────┬───────┘           │
│          ▼                          ▼                    │
│   ┌──────────────┐           ┌──────────────┐           │
│   │ 逐文档协作撰写 │──────┐    │ 加载文档注册表 │           │
│   └──────┬───────┘      │    └──────┬───────┘           │
│          ▼              │           ▼                    │
│   ┌──────────────┐      │    ┌──────────────┐           │
│   │ 写入文档 +    │      │    │ 分析文档影响   │           │
│   │ 更新注册表    │      │    └──────┬───────┘           │
│   └──────┬───────┘      │           ▼                    │
│          │              │    ┌──────────────┐           │
│          ▼              │    │ 确认并更新文档 │           │
│   docs/design/*.md      │    └──────┬───────┘           │
│   docs/review-state.json│           ▼                    │
│          │              │    ┌──────────────┐           │
│          │              └───>│ 更新状态文件   │           │
│          │                   └──────┬───────┘           │
│          │                          │                    │
│          └──────────────────────────┘                    │
│                  共享 review-state.json                   │
└─────────────────────────────────────────────────────────┘
```

两个技能通过 `docs/review-state.json` 中的文档注册表（`documents` 字段）实现状态共享：**初始化技能创建文档时写入注册表，审查技能读取注册表追踪文档并保持同步更新。**

## 技能列表

### project-init-docs — 项目文档初始化

```
了解项目 ──> 评估文档清单 ──> 用户确认范围 ──> 逐文档协作撰写 ──> 写入 docs/design/
    │                                                   │
    └────────────── 注册到 review-state.json ◄───────────┘
```

根据项目描述智能评估所需文档范围，与用户确认后协作撰写，输出到 `docs/design/` 目录。支持 PRD、TDD、ERD、API、SDD、项目计划等文档类型。

### code-review-doc-sync — 代码审查 + 文档同步

```
确定 review 范围 ──> 代码 Review ──> 输出报告
                                      │
                                      ▼
                              加载文档注册表（review-state.json）
                                      │
                                      ▼
                              分析代码变更对文档的影响
                                      │
                                      ▼
                              逐文档确认并更新 ──> 写入变更记录
                                      │
                                      ▼
                              更新 review-state.json
```

读取 git 提交差异进行代码审查，输出 review 报告，然后识别受影响的项目文档并提出更新建议，经确认后写入文档。通过 `docs/review-state.json` 跨会话记忆上次 review 位置。

## 结构

```
skills/
  <skill-name>/
    SKILL.md              # 技能定义（YAML frontmatter + Markdown）
    references/           # 参考资料（可选）
```

## 使用方式

将本仓库作为 Claude Code 的 skills 来源，技能会根据触发关键词自动匹配执行。

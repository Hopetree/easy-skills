# project-init-docs

项目文档初始化 Skill。

## 使用场景

- 开始一个新项目、需要创建项目文档
- 初始化项目规范文档、生成设计文档
- 询问项目需要哪些文档
- 用户说"我要开始一个新项目"

## 核心功能

1. **项目规模评估**：通过 2-3 句话了解项目核心功能、规模、技术复杂度，智能推荐所需文档范围
2. **文档清单推荐**：按四档规模（单人/多模块/多人协作/微服务）推荐不同文档组合（PRD、TDD、ERD、API、SDD、项目计划），每份附推荐理由
3. **逐文档协作撰写**：针对每种文档类型提出关键问题，基于用户回答生成初稿，迭代完善后写入 `docs/design/` 目录
4. **README 自动生成**：所有设计文档完成后，提取关键信息自动生成项目 README.md（项目简介、技术栈、文档索引、快速开始）
5. **文档注册表管理**：自动维护 `docs/review-state.json`，记录所有文档的路径、类型、描述和创建时间，与 code-review-doc-sync skill 共享状态
6. **中文优先**：所有文档使用中文撰写，遵循编号命名规范（如 `01_PRD_产品需求规格说明书.md`）

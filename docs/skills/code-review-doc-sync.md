# code-review-doc-sync

代码审查 + 文档同步 Skill。

## 使用场景

- 审查最近的 git 提交、检查代码变更
- review 某个 commit 范围的代码差异
- 代码变更后同步更新项目文档
- 使用 `--fix` 参数按 review 报告逐项修复问题
- 重新扫描项目文档、更新文档注册表

## 核心功能

1. **智能确定 Review 范围**：读取状态文件记忆上次 review 位置，自动计算增量范围，也支持用户手动指定
2. **多维度代码审查**：从代码质量、代码规范、潜在风险三个维度审查变更，输出面向 AI 的结构化报告（精确到行号 + 修改前后对比）
3. **文档影响分析**：根据变更文件类型自动判断哪些项目文档需要更新（API 变更 → 接口文档、数据库变更 → ERD 等）
4. **逐文档确认更新**：对受影响的文档生成更新建议，用户确认后写入，自动追加变更记录
5. **文档注册表管理**：维护 `docs/review-state.json`，统一管理所有项目文档的索引和状态
6. **`--fix` 修复模式**：读取 code-review 报告，按优先级逐项自动修复代码问题，实时更新报告状态（已修复 / 已忽略 / 待处理）

支持跨会话记忆上次 review 位置，不会重复审查已审过的提交。
